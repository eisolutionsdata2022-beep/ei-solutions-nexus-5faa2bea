# PSA Auto-ID Generation + Legacy Claim

## ID format
`RMPMCST-<10-digit-mobile>` (e.g. `RMPMCST-9876543210`) — matches the LEGACY portal exactly.
Generated by `generateVleId(uid, phone)` in `src/lib/pan-vle-id.ts`. If no phone is registered, falls back to a deterministic 10-digit FNV-1a hash of uid so the format stays uniform. Legacy claim accepts both `RMPMCST-…` and old `PSA…` IDs.

## Two ways to get a PSA ID
1. **Auto (new users)** — buy 2 successful `coupon-buy` transactions in PAN portal → `maybeGeneratePsaId({ uid, email, name, phone })` runs inside the success branch of `executePanService` and writes to `psa_ids/{uid}` atomically. `source: "auto"`.
2. **Legacy claim (existing members)** — Profile page → "I have an existing PSA ID" button opens a dialog. User pastes their old PSA ID, `claimLegacyPsaId({ uid, legacyPsaId, email, name, phone })` validates format (regex `^PSA[\dA-Z\-]{4,}$`) and writes to `psa_ids/{uid}` with `source: "legacy"`. No coupon threshold required. Idempotent (re-claim updates the same doc).

## Logic (`src/lib/psa-auto-id.ts`)
- `PSA_AUTO_THRESHOLD = 2`
- `countSuccessfulCouponPurchases(uid)` — counts `pan_transactions` where `serviceKey="coupon-buy"` AND `status="success"`. Refunded/failed excluded.
- `maybeGeneratePsaId(opts)` — Firestore `runTransaction` on `psa_ids/{uid}`: skips if exists or count < threshold; else writes new record with `source: "auto"`.
- `claimLegacyPsaId(opts)` — Firestore `runTransaction`; preserves original `generatedAt` if updating, sets `source: "legacy"`, adds `updatedAt`.
- `getPsaIdRecord(uid)` — direct doc read by uid.

## Trigger
`src/routes/retailer.pan-portal.tsx` → success branch when `service.key === "coupon-buy"` AND no `redirectUrl`. Passes `name` + `phone` from `appUser` (via dialog props `retailerName`, `retailerPhone`). On first generation: `🎉 Congratulations! Your PSA ID PSA######-MOBILE has been generated successfully.`

## UI surfaces
- **Retailer Dashboard** (`/retailer`) — green gradient congrats banner with PSA ID. Dismissible (localStorage `psa-banner-seen-{psaId}`). VLE ID card now shows `PSA######-<mobile>`.
- **Profile** (`/retailer/profile`) — PSA card with ACTIVE badge + "Migrated from old portal" tag if legacy. "Update PSA ID" / "I have an existing PSA ID" button → claim dialog (Malayalam + English copy).
- **Operator dashboard** (`/operator`) — VLE ID shows parent retailer's `PSA######-<mobile>`.
- **Admin Monitor** (`/admin/psa-ids`) — Status column shows ACTIVE + secondary badge `Auto` or `Migrated`.

## Firestore
- Collection: `psa_ids`, doc id = `uid` (one PSA per user)
- Schema: `{ uid, psaId, status:"active", generatedAt:ISO, successfulCouponCount, source:"auto"|"legacy", email, name, phone, updatedAt? }`
