# PSA Auto-ID Generation + Legacy Claim

## ID format
`RMPMCST-<10-digit-mobile>` (e.g. `RMPMCST-9876543210`) — matches the legacy UTI PSA portal exactly.
Generated by `generateVleId(uid, phone)` in `src/lib/pan-vle-id.ts`. If no phone is registered, falls back to a deterministic 10-digit FNV-1a hash of uid so the format stays uniform.

## Work logic (current)
1. **Auto-generate on first visit** — `ensurePsaIdRecord({ uid, email, name, phone })` runs on PAN Portal mount and writes `psa_ids/{uid}` with `source: "auto"`, `status: "active"`. Idempotent. User can IMMEDIATELY buy coupons using this ID.
2. **2-coupon milestone** — `successfulCouponCount` is refreshed from `pan_transactions` (status="success") on each ensure call. When count ≥ `PSA_ONBOARDED_THRESHOLD` (= 2), the UI shows the "fully onboarded" green banner and Active badge. The same VLE ID works on the official UTI PSA portal.
3. **Legacy claim** — Profile page → "I have an existing PSA ID" → `claimLegacyPsaId({ uid, legacyPsaId, ... })`. Validates format, writes with `source: "legacy"`. Overwrites the auto record.
4. **Provider-issued (optional)** — if "PSA ID Create" upstream call returns a different ID, `savePsaIdFromProvider()` overwrites with `source: "provider"`.

## Logic (`src/lib/psa-auto-id.ts`)
- `PSA_ONBOARDED_THRESHOLD = 2`
- `ensurePsaIdRecord(opts)` — Firestore `runTransaction` on `psa_ids/{uid}`: creates auto record if missing, refreshes coupon count on every call. Source: "auto".
- `savePsaIdFromProvider(opts)` — overwrites with provider-issued VLE ID. Source: "provider".
- `claimLegacyPsaId(opts)` — overwrites with user-pasted legacy ID. Source: "legacy".
- `countSuccessfulCouponPurchases(uid)` — counts `pan_transactions` where `serviceKey="coupon-buy"` AND `status="success"`.
- `getPsaIdRecord(uid)` — direct doc read.

## UI surfaces
- **PAN Portal hero** — VLE ID always visible. Badge: "Legacy linked" / "Provider issued" / `N/2 coupons` (auto, in progress) / "Active" (auto, fully onboarded).
- **Banner cards** — Blue "buy 2 coupons to fully activate" while count < 2; emerald "🎉 fully active" banner once milestone hit.
- **Profile** — PSA card with ACTIVE badge + "Migrated from old portal" tag if legacy. Update / claim dialog.
- **Admin Monitor** (`/admin/psa-ids`) — Status column shows ACTIVE + secondary badge `Auto` / `Provider` / `Migrated`.

## Firestore
- Collection: `psa_ids`, doc id = `uid`
- Schema: `{ uid, psaId, status:"active", generatedAt:ISO, successfulCouponCount, source:"auto"|"provider"|"legacy", email, name, phone, providerRef?, updatedAt? }`

## No coupon-buy gate
Coupon Buy is allowed immediately for all users — the auto-generated `RMPMCST-<mobile>` ID is sent upstream. (If upstream rejects with "Vle Data Not Exist", the user is prompted to use "PSA ID Create" or claim a legacy ID.)
