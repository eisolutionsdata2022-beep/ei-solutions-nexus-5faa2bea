# PSA Auto-ID Generation + Legacy Claim

## ID format
`PSA######-<10-digit-mobile>` (e.g. `PSA482917-9876543210`).
Generated by `generateVleId(uid, phone)` in `src/lib/pan-vle-id.ts` ‚Äî FNV-1a deterministic hash of uid ‚Üí 6 digits, suffixed with the user's registered mobile (last 10 digits, non-digits stripped). If no phone available, falls back to bare `PSA######`.

## Two ways to get a PSA ID
1. **Auto (new users)** ‚Äî buy 2 successful `coupon-buy` transactions in PAN portal ‚Üí `maybeGeneratePsaId({ uid, email, name, phone })` runs inside the success branch of `executePanService` and writes to `psa_ids/{uid}` atomically. `source: "auto"`.
2. **Legacy claim (existing members)** ‚Äî Profile page ‚Üí "I have an existing PSA ID" button opens a dialog. User pastes their old PSA ID, `claimLegacyPsaId({ uid, legacyPsaId, email, name, phone })` validates format (regex `^PSA[\dA-Z\-]{4,}$`) and writes to `psa_ids/{uid}` with `source: "legacy"`. No coupon threshold required. Idempotent (re-claim updates the same doc).

## Logic (`src/lib/psa-auto-id.ts`)
- `PSA_AUTO_THRESHOLD = 2`
- `countSuccessfulCouponPurchases(uid)` ‚Äî counts `pan_transactions` where `serviceKey="coupon-buy"` AND `status="success"`. Refunded/failed excluded.
- `maybeGeneratePsaId(opts)` ‚Äî Firestore `runTransaction` on `psa_ids/{uid}`: skips if exists or count < threshold; else writes new record with `source: "auto"`.
- `claimLegacyPsaId(opts)` ‚Äî Firestore `runTransaction`; preserves original `generatedAt` if updating, sets `source: "legacy"`, adds `updatedAt`.
- `getPsaIdRecord(uid)` ‚Äî direct doc read by uid.

## Trigger
`src/routes/retailer.pan-portal.tsx` ‚Üí success branch when `service.key === "coupon-buy"` AND no `redirectUrl`. Passes `name` + `phone` from `appUser` (via dialog props `retailerName`, `retailerPhone`). On first generation: `üéâ Congratulations! Your PSA ID PSA######-MOBILE has been generated successfully.`

## UI surfaces
- **Retailer Dashboard** (`/retailer`) ‚Äî green gradient congrats banner with PSA ID. Dismissible (localStorage `psa-banner-seen-{psaId}`). VLE ID card now shows `PSA######-<mobile>`.
- **Profile** (`/retailer/profile`) ‚Äî PSA card with ACTIVE badge + "Migrated from old portal" tag if legacy. "Update PSA ID" / "I have an existing PSA ID" button ‚Üí claim dialog (Malayalam + English copy).
- **Operator dashboard** (`/operator`) ‚Äî VLE ID shows parent retailer's `PSA######-<mobile>`.
- **Admin Monitor** (`/admin/psa-ids`) ‚Äî Status column shows ACTIVE + secondary badge `Auto` or `Migrated`.

## Firestore
- Collection: `psa_ids`, doc id = `uid` (one PSA per user)
- Schema: `{ uid, psaId, status:"active", generatedAt:ISO, successfulCouponCount, source:"auto"|"legacy", email, name, phone, updatedAt? }`
